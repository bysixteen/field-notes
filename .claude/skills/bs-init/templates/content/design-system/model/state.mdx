## Definition

State is the **interaction stage** a component is in. State transitions are visual; the underlying sentiment and emphasis don't change.

## Values

| Value | Meaning |
|-------|---------|
| `rest` | Default — not being interacted with. |
| `hover` | Pointer is over the component. |
| `active` | Component is being pressed / activated. |
| `disabled` | Component is non-interactive. |

Replace this list with the states your project uses. Some systems add `selected`, `focus-visible`, `loading`. Add only what you have visual treatments for.
