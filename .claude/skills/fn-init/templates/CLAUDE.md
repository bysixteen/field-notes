# {{project_name}}

## Context stack

This project follows the bysixteen four-layer agent context stack. Each layer answers a different kind of question; the layers compose so an agent can route from a small, always-loaded entry point down to the source of truth without dumping the whole system into context.

- **Layer 1 — `CLAUDE.md` (this file)**: identity, conventions, links into deeper layers.
- **Layer 2 — `DESIGN.md`**: compact design-system reference. Generated by `fn-design-md --from-dimensional . --out . --name {{project_name}}` from `tokens.json` + `components.json` + `content/design-system/model/*.mdx`. Re-run after any dimensional change.
- **Layer 3 — `tokens.json`**: full DTCG 2025.10 token source. Query specific tokens; never load whole.
- **Layer 4 — per-component contracts**: hand-authored under each component's directory. Captures what the dimensional model can't enforce (legal value combinations, layout invariants, ARIA roles).

## Skills

The bysixteen design-system toolkit ships as the [`field-notes-toolkit`](https://github.com/bysixteen/field-notes/tree/main/field-notes-toolkit) Claude plugin. Once installed, all `fn-*` skills (`fn-design-md`, `fn-init`, `fn-tokens`, `fn-review`, etc.) are reachable from a Claude session, plus `start-issue` and `start-work` workflows.
