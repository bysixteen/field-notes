## Definition

Size scales a component's dimensions (height, padding, font-size) in proportion. Sizes do not change the component's meaning — only its physical presence.

## Values

| Value | Meaning |
|-------|---------|
| `xs` | Smallest — for dense interfaces. |
| `sm` | Compact. |
| `md` | Default. |
| `lg` | Roomy. |
| `xl` | Largest — for hero CTAs. |

Replace this list with the sizes your project actually ships. Three sizes (sm, md, lg) is a common minimum.
