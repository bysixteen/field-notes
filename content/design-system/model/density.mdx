## Definition

Density controls how content is **distributed within** a component's footprint: the internal padding and gap values that cluster or spread the component's content.

Density answers: *how tightly is the content packed inside?*

## Values

| Value | Meaning |
|-------|---------|
| `default` | Standard internal spacing |
| `relaxed` | More generous internal padding and gaps |

**Default:** `default`

## When to Use

Density is a **layout/environment** decision. It adapts components to the context they're placed in:

- Data-dense interfaces (tables, dashboards) → `default` density
- Content-focused layouts (articles, cards) → `relaxed` density
- Touch interfaces with generous spacing → `relaxed` density

## Common Mistake: Using Density When You Mean Size

If a component "feels too big" in a tight layout (e.g. buttons in a data table), ask which problem you're actually solving:

- **The component takes up too much vertical/horizontal space** → change `size` (e.g. `size="sm"` for a smaller touch target and height)
- **The component's content feels loosely arranged inside its bounds** → change `density` (e.g. `density="default"` for tighter internal packing)

Density adjusts **internal arrangement**. Size adjusts **the box itself**.

## In Tokens

Density affects internal spacing tokens — padding, gap, and sometimes line-height within components. It does not affect the component's outer dimensions (that's size).

## Relationship with Size

A large component with tight density clusters its content centrally, leaving proportionally more negative space at the edges. A small component with relaxed density spreads its content out, using more of its compact footprint for internal breathing room.

Neither dimension substitutes for the other. See [Density vs Size](/design-system/composition/density-vs-size) for worked examples.
