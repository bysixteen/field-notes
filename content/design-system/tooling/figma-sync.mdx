## Overview

Two shell scripts synchronise design tokens with Figma through the [PRISM](/design-system/prism) MCP bridge:

| Script | Direction | Purpose |
|--------|-----------|---------|
| `figma-push.sh` | Local → Figma | Push generated token collections into Figma as variables |
| `figma-pull.sh` | Figma → Local | Pull anchor variables from Figma into `generated/figma-anchors.json` |

Both scripts read connection configuration from `.mcp.json` at the project root and call Prism tools by canonical name through the `scripts/_prism-call.mjs` helper. Nothing is hardcoded — `figma.fileId` is resolved at runtime, never embedded in the scripts.

---

## Prerequisites

- **jq** — JSON processor (`brew install jq`)
- **node ≥ 22** — runs the `_prism-call.mjs` helper
- **Prism MCP plugin** — running and connected to Figma. The plugin must be open with the WebSocket bridge listening on `ws://localhost:7890`.
- **`.mcp.json`** — project-root config containing the Prism server entry and the Figma file ID.

### `.mcp.json` structure

The canonical shape — values are project-specific. See [PRISM](/design-system/prism) for the full configuration reference.

```jsonc
{
  "mcpServers": {
    "prism": {
      "command": "node",
      "args": ["/path/to/prism/dist/mcp/index.js"],
      "env": {
        "FIGMA_PAT": "<your-personal-access-token>",
        "FIGMA_MCP_PORT": "7890"
      }
    }
  },
  "figma": {
    "fileId": "<file-id>"
  }
}
```

> **v0.3.0+:** Once Prism is published to npm, replace `command`/`args` with `"command": "npx", "args": ["-y", "prism-figma-mcp"]`.

## figma-push.sh

Pushes locally-generated token JSON to Figma as variable collections.

### Usage

```bash
# Push from the default input file
./scripts/figma-push.sh                              # uses generated/figma-variables.json

# Push from an explicit file
./scripts/figma-push.sh path/to/variables.json

# Push from stdin
cat variables.json | ./scripts/figma-push.sh

# Dry run — show planned tool calls and payload, send nothing
./scripts/figma-push.sh --dry-run
```

### Input format

The input JSON follows the Prism push payload schema. The script injects `fileId` from `.mcp.json` automatically.

```json
{
  "collections": [
    {
      "name": "Primitives",
      "modes": ["Light", "Dark"],
      "variables": [
        {
          "name": "sky-50",
          "type": "COLOR",
          "valuesByMode": {
            "Light": "#f0f9ff",
            "Dark": "#082f49"
          }
        }
      ]
    }
  ]
}
```

### Tools invoked (Prism, by canonical name)

`check_connection`, `set_target_file`, `get_variable_collections`, `get_collection_variables`, `create_variables_batch`, `update_variables_batch`.

### Preflight checks

Before pushing, the script verifies:

1. `jq` is installed
2. `node` is on PATH
3. `.mcp.json` exists at the repo root
4. `scripts/_prism-call.mjs` helper exists
5. `.mcp.json` contains `figma.fileId`
6. Prism `check_connection` returns ok
7. Prism `set_target_file` accepts the configured `fileId`

If any check fails the script exits with a descriptive error.

## figma-pull.sh

Pulls anchor variables from Figma and writes them to `generated/figma-anchors.json`.

### Usage

```bash
# Pull the default "Anchors" collection
./scripts/figma-pull.sh

# Override the collection name
./scripts/figma-pull.sh --collection Primitives

# Dry run — show planned tool calls, fetch nothing
./scripts/figma-pull.sh --dry-run
```

### Output

The script writes exclusively to `generated/figma-anchors.json`. It never modifies `.ts` source files. The output follows the Prism pull-response shape:

```json
{
  "collection": "Anchors",
  "modes": ["Light", "Dark"],
  "variables": [
    {
      "name": "accent",
      "type": "COLOR",
      "valuesByMode": {
        "Light": "#2563eb",
        "Dark": "#60a5fa"
      }
    }
  ]
}
```

### Tools invoked (Prism, by canonical name)

`check_connection`, `set_target_file`, `get_variable_collections`, `get_collection_variables`.

### Preflight checks

Before pulling, the script verifies:

1. `jq` is installed
2. `node` is on PATH
3. `.mcp.json` exists at the repo root
4. `scripts/_prism-call.mjs` helper exists
5. `.mcp.json` contains `figma.fileId`
6. Prism `check_connection` returns ok
7. The named collection (default `Anchors`) exists in the file

## Tool calling

Both scripts call Prism tools by canonical name through the `scripts/_prism-call.mjs` helper — there is no runtime tool discovery. The helper opens a WebSocket to `ws://localhost:7890` (configured via `FIGMA_MCP_PORT` in `.mcp.json`) and invokes `tools/call` with the named tool.

The standard preamble for every script run is:

```
prism_call check_connection {}
prism_call set_target_file { "fileId": "<from .mcp.json>" }
```

After this preamble, the script issues the operation-specific batched calls (push: `create_variables_batch` and `update_variables_batch` per collection; pull: `get_variable_collections` then `get_collection_variables`).

If the bridge is unreachable, `check_connection` fails first — verify the Prism plugin is open in Figma and the WebSocket bridge is on `ws://localhost:7890`.

## Error handling

All errors are prefixed with `[prism]` for easy identification in logs.

| Error | Cause | Resolution |
|-------|-------|------------|
| `[prism] jq not found (brew install jq)` | jq not installed | `brew install jq` |
| `[prism] node not found (need node ≥ 22)` | node not on PATH or version too old | Install node ≥ 22 |
| `[prism] .mcp.json missing at <path>` | Missing config file | Create `.mcp.json` at project root |
| `[prism] helper missing: scripts/_prism-call.mjs` | Helper deleted or repo incomplete | Restore from git |
| `[prism] figma.fileId not set in .mcp.json` | Config missing file ID | Add `figma.fileId` to `.mcp.json` |
| `[prism] input not found: <path>` | Push input file missing | Create input or use stdin |
| `[prism] input is not valid JSON` | Malformed input | Validate with `jq . <file>` |
| `[prism] collection "<name>" not found in file <fileId>` | Pull target missing | Create collection in Figma, or pass `--collection` |

Connection-level failures (`check_connection`) surface from the helper with the underlying WebSocket error — first thing to check is that the Prism plugin is running in the Figma desktop app.
