These are not theoretical guidelines. They are principles distilled from real project delivery — patterns that emerged from mistakes made, lessons learned, and workflows that proved their worth across multiple engagements. Each one compounds: the more consistently they are applied, the more value they generate over time.

## Core Principles

| Principle | Summary |
|---|---|
| [Distill, Don't Repeat](/principles/distill-dont-repeat) | The output of a sprint is synthesized knowledge, not raw artifacts. Create living documents that eliminate the need to re-read source material. |
| [Clarity, Attribution, Reversibility, Consistency](/principles/clarity-attribution-reversibility-consistency) | The four pillars of validation for any prototype or feature. Every piece of work must pass all four checks. |
| [Compounding Engineering](/principles/compounding-engineering) | Build guidance from real mistakes, not upfront speculation. Let documentation grow organically from actual needs. |
| [Verification Over Instruction](/principles/verification-over-instruction) | Give AI a way to verify its work rather than a long list of instructions. Success conditions beat specifications. |
| [Dissent is a Feature](/principles/dissent-as-feature) | Structurally encourage and document dissent. Record overruled concerns for future reference. |
| [Answer First](/principles/answer-first) | Every document must have the conclusion in the first body section. Never bury the lead. |

## Frameworks

These frameworks operationalize the principles above into repeatable systems.

### The Delivery Framework

A project-agnostic operating system for design and delivery. It covers the full lifecycle from initial audit through sprint execution and journey mapping, providing a canonical structure for repository architecture, team composition, and validation workflows.

[Read the Delivery Framework](/principles/delivery-framework)

### Project Squad

The portable toolkit that implements the Delivery Framework. It provides personas, sprint commands, and living document templates for any project.

[Read about Project Squad](/principles/project-squad)

### Synthesis & Validation Framework

A three-artifact model and four-pillar validation system for preventing context burn. It defines the exact documents to maintain, the ritual for updating them after each sprint, and the checklist for validating every piece of work.

[Read the Synthesis & Validation Framework](/principles/synthesis-validation)

### Journey Mapping Template

A reusable structure for mapping any part of a system from user need to validated outcome, including a complete markdown template and worked example.

[Read the Journey Mapping Template](/principles/journey-template)

The step-by-step bootstrap sequence that ties together Project Squad, Field Notes, and the Delivery Framework into a single startup workflow.
