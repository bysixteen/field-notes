## 3. Needs human review

### R1. Duplicate / drifting skill scripts (the headline finding)

`.claude/skills/fn-design-md/scripts/` and `field-notes-toolkit/skills/fn-design-md/scripts/` are **partial duplicates with active drift**. The toolkit copy is invoked through the shim at `field-notes-toolkit/scripts/emit-design-md.mjs:10`. `pack-plugin.mjs` is the intended sync mechanism but currently nothing prevents the two from diverging — recent commit `0252ad5` cleaned the *outer* duplicate at `field-notes-toolkit/scripts/` but missed this inner cluster.

**Canonical: `.claude/skills/fn-design-md/scripts/`.** Evidence: every file the toolkit has exists in `.claude/`; 9 files exist *only* in `.claude/`; the 2 shared files that disagree are both larger in `.claude/`. The 4 unique-to-`.claude/` lib files (`template.mjs`, `redact.mjs`, `thin-index.mjs`, `sections.mjs`) match the recent feature commits `edc951714` (per-section markers, redaction halt), `cfc5a6e` (thin-index migration), and `2aa3187` (tokenNamespace wiring) — all 2026-04-27..29. Recent work consistently lands in `.claude/` first; the toolkit lags.

**Recommended fix shape: `.claude/` is the single source of truth; the toolkit copy becomes a generated artifact.** Concretely:

1. Add `field-notes-toolkit/skills/fn-design-md/scripts/` to `.gitignore`.
2. `pack-plugin.mjs` copies (or symlinks at pack time, stored in `dist/`) `.claude/skills/fn-design-md/scripts/` into the toolkit tree before producing the `.plugin` artefact.
3. Consumers get the canonical implementation from the published `field-notes-toolkit` package; nobody commits the duplicate.

A rsync-at-pack-time without removing the committed copy is the worst of both worlds — it leaves a window where the trees can diverge between commits, which is exactly how this state was reached.

**No pre-migration guard.** A static `diff -r` CI gate was originally suggested as a "stop the bleeding" measure, but it cannot function pre-R1: the trees on main are already divergent, so any PR touching either tree fails the gate regardless of whether it introduces new drift or sits adjacent to pre-existing drift. PR #212 was opened with this approach and withdrawn (see [its close comment](https://github.com/bysixteen/field-notes/pull/212)). The natural place for a regression check is *after* R1 collapses the trees — either folded into PR 10 itself or skipped entirely once the toolkit copy is gitignored and divergence is structurally impossible.

Drop `scripts` from the exclude array.
